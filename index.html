Project Overview

Adaptive Speculative Decoding (AdaSpec) is an advanced research framework for accelerating text generation in Large Language Models (LLMs). Instead of classic autoregressive generation, a fast small model (drafter) prepares a draft of several tokens, and a large model (target) verifies them in a single pass. If the tokens are correct, we get multiple tokens for the price of one large model forward pass.

The framework resolves three fundamental challenges of speculative decoding: cross-vocabulary mismatch, inability to adapt models on-the-fly, and static draft lengths.

System Architecture

Key Components & Extensions

Core Loop

Speculative Decoding

The fundamental step-by-step cycle implemented in the SpeculativeDecoder class.

  • Drafting: Autoregressive generation of k tokens using KV cache (saving O(seq_len) compute).
  • Verification: Parallel verification of k+1 tokens by the target model.
  • Accept/Reject: GPU-vectorized evaluation of the min(1, p/q) condition.
  • Bonus Token: Sampling a token from the residual distribution at the rejection point to guarantee correctness.
Translation

Cross-Vocabulary Translation

A multi-tier system bridging models with different tokenizers (e.g., Llama and Qwen).

  • Rule 1 (Exact match): Exact 1-to-1 probability transfer for matching IDs (covers ~70%).
  • Rule 2 (Approximate): Substring matching via Aho-Corasick automaton and probability redistribution.
  • Tokenizer Lattice: DP-based graph for exact replacement of Rule 2 (currently in research).
Learning

Online Distillation

Continuous fine-tuning of the drafter model during generation via LoRA with gradient accumulation.

  • KL Divergence: For tokens with direct Rule 1 mapping.
  • N-gram NLL: For merged tokens on accepted sequences.
  • Contrastive Loss (InfoNCE): Using rejected tokens as hard negatives.
Dynamic K

Adaptive Draft Selection

Intelligent control of the draft length (k) based on context difficulty (heuristic and learned).

  • Speedup Predictor: Predicts the optimal k based on hidden states.
  • Latent Regime Controller: HMM identifying difficulty regimes (easy, normal, hard) and sampling k from a truncated Poisson distribution.
Routing

Multi-Drafter Routing

Dynamic selection of the best drafter model from a pool for a specific prompt (trade-off speed vs quality).

  • Bandit Algorithms: UCB1, Thompson Sampling, and LinUCB for on-the-fly policy learning.
  • MLP Router: Prediction based on prompt embeddings (length, vocabulary diversity).
Cache & Universal

Cache & Universal Drafter

Modules for reducing computational overhead and enabling universality.

  • N-gram Cache: 65K-entry cache with LRU, LFU, and Acceptance-weighted strategies to bypass drafter forward passes.
  • Universal Drafter: A single drafter model (via target-specific LoRA adapters) serving different target families.

Critical Optimizations (Performance Tweaks)

The project underwent a series of low-level performance enhancements making theoretical algorithms efficient in practice.

KV Cache Reuse (Drafter)

Key-value cache is preserved across steps, processing only 1-2 new tokens instead of a full O(seq_len) pass. Yields 10-60x speedup.

GPU-Vectorized Accept/Reject

Eliminated Python loops with CPU/GPU synchronizations. Acceptance condition is now computed with a single `cumprod` on GPU.

Bonus Token Forward Reuse

A single drafter forward pass extends the KV cache and outputs hidden states for the Adaptive controller (saves 1 redundant pass).

Dense Transfer Matrix

Dense matmul is used instead of sparse kernel launch overhead for small vocabularies, significantly speeding up Rule 2 mapping.
